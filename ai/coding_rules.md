# Coding Standards

General principles:

- Prefer simplicity over cleverness.
- Functions should remain small and single-purpose.
- Avoid embedding business logic in API routes.
- Ensure all external inputs are validated.
- Maintain clear separation between API, service, and model layers.

Python:

- Follow PEP8 style guidelines.
- Use explicit variable names.
- Avoid global state.

Frontend:

- Use TypeScript interfaces for API responses.
- Keep API logic separated from UI components.
